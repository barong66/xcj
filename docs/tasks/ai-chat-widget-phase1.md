## Summary

Full backend implementation for the AI Chat Widget feature. After this phase, the chat API is functional and testable via curl. No frontend changes required to verify.

**Verification:** `curl "http://localhost:8080/api/v1/chat/config?slug=unknown" -H "X-Forwarded-Host: localhost:3000"` returns `{"data":{"enabled":false},"status":"ok"}`.

## What Was Built

### Database (migration 017)

File: `scripts/migrations/017_chat_settings.sql`

Three new columns on `accounts`:
- `chat_enabled BOOLEAN NOT NULL DEFAULT false` — enables the chat widget per account
- `chat_prompt TEXT` — optional custom system prompt; auto-generated from account metadata if NULL
- `chat_ad_text TEXT` — optional ad copy injected by Grok as a CTA at the end of chat responses

### Config

File: `api/internal/config/config.go`

Added `XAIAPIKey string` field loaded from `XAI_API_KEY` env var.

### ChatStore

Files: `api/internal/store/chat_store.go`, `api/internal/store/helpers.go`

`GetChatConfig(ctx, siteID, slug)` — queries accounts joined with site_videos and categories. Returns `nil` if account not found, inactive, or `chat_enabled=false`. Returns `*ChatConfig` with: `AccountID`, `Name`, `Username`, `Country`, `Categories`, `SocialLinks`, `ChatPrompt`, `ChatAdText`.

### ChatHandler

File: `api/internal/handler/chat.go`

Two handlers:

**`GET /api/v1/chat/config?slug={slug}`**
- Returns `{enabled, model_name, greeting}`
- If `chat_enabled=false` or account not found: returns `{enabled: false}`
- Greeting is generated by a single non-streaming Grok call (80 tokens), cached in Redis with 1h TTL
- Grok model: `grok-3-mini`

**`POST /api/v1/chat/message`** (SSE)
- Body: `{"slug": "...", "message": "..."}`
- Builds system prompt from `chat_prompt` (verbatim) or auto-generates from account data
- Streams Grok response as SSE: `data: {"delta":"..."}` events
- Extracts `[CTA text|url]` tag from Grok output, sends final `data: {"done":true,"cta":{"text":"...","url":"..."}}`

### Routes

Files: `api/internal/handler/router.go`, `api/cmd/server/main.go`

- `GET /api/v1/chat/config` and `POST /api/v1/chat/message` registered
- `xaiAPIKey` passed as parameter to `NewRouter()`
- `WriteTimeout: 0` set on `http.Server` to support long-lived SSE connections

### Account Model

File: `api/internal/model/account.go`

Added `ChatEnabled bool` to `Account` struct. Updated `GetBySlug` and `GetByID` queries to include the 3 new chat columns.

### Admin Store

File: `api/internal/store/admin_store.go`

Extended `UpdateAccountInput` with `ChatEnabled *bool`, `ChatPrompt *string`, `ChatAdText *string`. Extended `AdminAccount` struct with the same fields. `UpdateAccount()` handles each field individually. `getAccountByID` SELECT updated to include new columns.

### Analytics

File: `api/internal/handler/event.go`

Added to `validEventTypes`:
- `chat_open`
- `chat_message`
- `chat_cta_click`
